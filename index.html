Kanıt 1

Hız (Performance)

Biz sadece hızlı değiliz — native hızdayız. Zet bir sanal makine (VM) kullanmaz. Yazdığınız Zet kodu, dünyanın en optimize derleyicisi olan Rust/LLVM altyapısına çevrilerek doğrudan native binary üretilir.

Yaptığımız CPU-bound testlerde (fib(40)):

Go (GC Overhead) ~444 ms
Zet (No GC) ~249 ms

Bu fark bir optimizasyon numarası değil; GC'siz bellek modeli ve saf fonksiyon analizinin doğal sonucudur.

// Zet Benchmark — fib(40)
deterministic fn fib(n: i64) -> i64 {
  if n <= 1 { return n }
  return fib(n-1) + fib(n-2)
}
// Leke Analizi (Compile-Time)
nondeterministic fn login(user: Untrusted) -> Void {

  // ❌ DERLEME HATASI: Untrusted veri validate edilmeden kullanılamaz
  call Database.query(user)

  validate user {
    success: {
      // ✅ user artık güvenli (String)
      call Database.query(user)
    }
  }
}
Kanıt 2

Zero Trust — Leke Analizi

Diğer dillerde şu soruyu sen sorarsın: "Bu veri güvenli mi?"
Zet'te bu soruyu derleyici sorar.

Zet'in Leke Analizi (Taint Analysis) motoru:

  • Dış dünyadan gelen veriyi Untrusted olarak işaretler.
  • Bu veri validate edilmeden DB'ye gidemez, Network'e çıkamaz.
  • Türetilen veriler de (json, index) lekeyi miras alır.

Bu bir uyarı değildir. Derleme hatasıdır.
Zet ile yazılmış bir kodda "Yanlışlıkla güvensiz kod yazdım" deme ihtimali yoktur.

Kanıt 3

Akıllı Motor (Smart Codegen)

Zet derleyicisi, kodunuzu anlar.

  • Saf Matematik (Deterministic): Async overhead tamamen kaldırılır. Senkron, saf makine kodu üretilir.
  • I/O İşlemi (Nondeterministic): Otomatik olarak async/await moduna geçilir. Sen async yazmakla uğraşmazsın, Zet senin yerine karar verir.
  • call Keyword: Nondeterministik fonksiyona call kullanmazsan — derleme hatası. Deterministik fonksiyona call kullanırsan — derleme hatası.
// Smart Codegen Örneği
deterministic fn mat() -> i64 {
  // -> No Async Overhead
  return 2 + 2
}

nondeterministic fn api() -> Void {
  // -> Auto Async/Await
  call HTTP.get("url")
}
// Structured Concurrency
nondeterministic fn main(args: Untrusted) -> Void {

  // ❌ DERLEME HATASI: spawn scope dışında kullanılamaz
  spawn DB.log("yasak!")

  scope Islerim {
    // ✅ spawn sadece scope içinde
    spawn veri_getir()
    spawn DB.log("paralel!")
  }
  // Scope kapandı — tüm işçiler bitti, zombie yok.
}
Kanıt 4

Structured Concurrency

Diğer dillerde spawn veya go çağırırsın ve umudu bırakırsın.
Zet'te paralel iş sadece scope içinde başlatılır.

  • spawn scope dışında kullanılamaz — derleme hatası.
  • Scope kapanmadan program ilerlemez — tüm JoinHandle'lar await edilir.
  • Zombie process, orphan thread oluşması yapısal olarak imkânsızdır.

Concurrency güvenliği, geliştirici disiplinine değil — derleyici kurallarına bağlıdır.

Sistem Mimarisi ve Detaylar

01. Sözdizimi (Syntax)

Mevcut Durum (v0.2)

  • Değişkenler: let x = 10 ile tanımlanır.
  • Tipler: i64 (Tamsayı), String (Metin), Untrusted (Kirli Veri), Void, Array<T>.
  • Kontrol: if/else, while ve aralıklı for (0..10) döngüsü.
  • Fonksiyonlar: deterministic fn ve nondeterministic fn zorunlu ayrımı.
  • Güvenlik: validate bloğu, call keyword — derleyici zorlar.

Planlanan (v0.3+)

  • Boolean: true/false literal var ama tip olarak i64 (0/1) kullanıyoruz.
  • Float: Yok. Bilimsel hesaplama için şart.
  • Mutation: liste[0] = 5 diyemiyoruz. Bunu yaparsak tam bir dil oluruz.

02. Bellek & Scope

Bellek Yönetimi

GC Yok: Garbage Collector kullanmıyoruz. Değişken scope'un sonuna (}) geldiği an Rust'ın Drop mekanizması devreye girer.

Manuel Yok: malloc veya free yok. Derleyici halleder.

Structured Concurrency (v0.2 ✅)

JoinHandle: Scope içindeki her spawn bir JoinHandle üretir. Scope kapanmadan önce tüm handle'lar await edilir.

Zombie Process İmkânsız: spawn scope dışında kullanılamaz — derleyici reddeder. Scope kapanmadan program ilerlemez.

Validate Mekanizması (v0.2 ✅)

Untrusted Tipi: Dışarıdan gelen veri Untrusted tipindedir. Derleyicinin Taint Analysis motoru, derleme zamanında tüm akışı tarar. Eğer Untrusted bir veri, validate edilmeden herhangi bir yere giderse derlemeyi reddeder. Türevleri de (json, index) lekelenir.

v0.2'de: Validate bloğu gerçek match Ok/Err kodu üretir. Başarılı dalda değişken String (Trusted) olur.

Determinizm Analizi (v0.2 ✅)

Derleyici Kontrolü: deterministic fonksiyon içinde I/O, HTTP, Console çağrısı yapılamaz — derleme hatası.

call Keyword: Nondeterministik fonksiyona call kullanmak zorunlu. Deterministik fonksiyona call kullanmak yasak. İkisi de derleme zamanında zorlanır.
